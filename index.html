    Schema for First Past the Post (FPP) election results.

    This is an election results schema for publishing election results in an open fashion, it is designed for the First Past the Post electoral model. The GitHub repository contains files to help you get started, while the gh-pages branch contains the webpage.

    Contribution

    If you have any suggestions on how to make this schema better PLEASE contribute them either by making a pull request with your changes publish on the gh-pages branch, or by posting suggestions in the GitHub bug tracker.

    Tables

    Name Details
    Candidates One or more people who are running in a given jurisidiction in a given election.
    Results Vote counts for each candidate in each jurisdiction in each election.
    Jurisdictions An areas in which a candidate will represent.
    Polls An area contained within a jursdiction which is assigned to a specific polling location.
    Elections One or more elections that are provided.
    Election Jurisdictions The jurisdictions which are used in each election.
    Affiliations One or more political affiliations that are represented in any election.

    Candidates

    Field Name Required Details
    candidate_id Required, Unique A unique identifier to represent this candidate.
    candidate_name Required The name used by the candidate to register for the election.
    affiliation_id Optional The political party that this candidate is representing. If empty or null the candidate should be regarded as Independent.

    Results

    Field Name Required Details
    counted_for Required The candidate_id representing a candidate OR one of: REJECTED, UNMARKED, DECLINED.
    jurisdiction_id Required The jurisdiction where these results were collected.
    poll_id Required The poll in which these results were collected from.
    votes Required The amount of votes that were counted.

    Jurisdictions

    Field Name Required Details
    jurisdiction_id Required, Unique A unique identifier for this area.
    boundary Required A geographical object representing the boundary of the jurisdiction.
    name Required The name of the jurisdiction.
    total_registered Optional The total number of registered voters in the jurisdiction.

    Polls

    Field Name Required Details
    jurisdiction_id Required The jurisidiction_id of the jurisdiction that this poll is contained within.
    poll_id Required, Unique A unique identifier for this polling location.
    boundary Required A geographical object representing the boundary of the poll.
    name Optional The name of the jurisdiction.
    total_registered Optional The total number of registered voters in the poll.

    Elections

    Field Name Required Details
    election_id Required, Unique A unique identifier for the election.
    name Required The registered name of the election.
    date Required The date that voting was held for the election.

    Election Jurisdictions

    Field Name Required Details
    jurisdiction_id Required, Unique A unique identifier for this area.
    election_id Required The election_id for which this jurisdiction is used in.

    Affiliations

    Field Name Required Details
    affiliation_id Required, Unique A unique identifier for this political party/affiliation.
    long_name Required The full name or registered name of the political party/affiliation.
    short_name Optional The short name of the political party/affiliation.
    abbreviation Optional The abbreviation of the political party/affiliation.
